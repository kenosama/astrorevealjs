Astro is a framework

it has those features:
  • is a SSR system for building website
  • Works with component (from astro, react ect...)

  • Allows to load Js-involved script on load/view. wich allows faster rendering of the page.

SSR?

it means the server renders and caches the final project, it allow render of the project on the server side, and then send it to the client.

Routing?

Astro uses a simple routing system. One astro file means one page, for example, the index.astro file means "mysite.com/", the about.astro file means "mysite.com/about", and so on.

A folder in the pages folder means a dynamic route, for example, the folder "pages/blog" means "mysite.com/blog/*", the * means that the page can be anything, for example, "mysite.com/blog/first-post" or "mysite.com/blog/second-post".

Installation

installation is easy with npm, just run 
npm create astro@latest
you can run it with 
npm run dev
integrate frameworks for css, ui, and more with for example 
npx astro add react tailwind partytown
and you're good to go 🚀.

folder structure

  • the components folder contains all the components
  • the layouts folder contains all the layouts
  • the pages folder contains all the pages
  • the public folder contains all the static files
  • the styles folder contains all the styles
  • the utils folder contains all the utils

Deployment

For a simple static site, you can build your site with 

npm run build
and then deploy it on the server of your choice.

for automated build and deployment, you can use vercel, netlify, or any other service that supports ssr with astro. they will automatically build and deploy your site when you push to your git repository. in my case i used netlify to deploy my site.
